# China Stock Market Pro — 设计白皮书

> 为什么这个 Skill 是这样设计的？用户用它能得到什么？和同花顺/东方财富/Yahoo Finance 有什么不同？

---

## 一、文件结构与设计理由

```
china-stock-market-pro/
├── SKILL.md          (366 行) ← Claude 的行为说明书
├── README.md         (136 行) ← 给人看的项目介绍
├── DESIGN.md         (本文件) ← 设计理念
├── .gitignore
└── scripts/
    ├── cnstock       (1667 行) ← 核心引擎
    └── portfolio.py  (490 行)  ← 组合管理
```

### 1. SKILL.md — Claude 的"行为说明书"

**是什么**：一份用 Markdown 写的系统指令。当用户通过 Claude Code 调用这个 skill 时，Claude 会把 SKILL.md 的内容作为"操作手册"加载到上下文。

**为什么这样设计**：
- **不是给用户看的**，是给 Claude 看的。里面写的是"当用户说X，你应该做Y"——这是 prompt engineering，不是产品文档。
- **固定输出模板**（119-169行）：强制 Claude 按"行情→基本面→走势→小白解读"的结构输出，而不是随意丢数据。为什么？因为新手面对一堆数字会焦虑，固定结构=可预期=放心。
- **Multi-Stock Workflow**（171-239行）：告诉 Claude "用户说多只股票时先问对比还是建组合"。这是用户体验设计——不做猜测，让用户做主。
- **Limitations 章节**（254-295行）：这是关键。告诉 Claude 哪些数据可能不准、哪些市场休市、哪些图表有字体问题。目的是让 Claude 在出报告时主动给用户打预防针，而不是用户踩坑后来骂。
- **Error Handling 章节**（299-335行）：分两层——脚本层（Python 怎么处理）和 AI 层（Claude 遇到错误应该怎么跟用户说）。双保险。

**竞争点**：stock-market-pro 的 SKILL.md 只有 46 行，没有输出模板，没有局限说明，没有错误处理。所以 Claude 用那个 skill 时就是"把命令输出甩给你自己看"，用这个 skill 是"整理好、解释给你听"。

### 2. scripts/cnstock — 核心数据引擎

**是什么**：一个 1667 行的 Python 单文件 CLI 工具。7 个子命令：`price` / `chart` / `compare` / `fundamentals` / `earnings` / `history` / `market`。

**为什么不是微服务架构而是单体脚本**：
- 它是 Claude Code skill 的"可执行文件"，不是独立产品。uv 的 `--script` 模式要求一个入口文件就能跑，依赖声明写在文件头部（PEP 723 inline metadata）。
- 单体脚本 = 用户 clone 后一行命令就跑，不需要 `pip install`、不需要 `setup.py`、不需要虚拟环境。

**多数据源降级链**（最核心的设计决策）：
```
美股: Yahoo Finance → Stooq → Yahoo HTTP 直连
A股: 新浪财经 → 腾讯财经 → baostock → AKShare
```
每一层都有独立的 fetcher 函数，一个挂了静默跳到下一个。为什么？因为这些数据源都是免费的、不稳定的。新浪偶尔封 IP，Yahoo 经常 429，AKShare 可能被代理拦截。单数据源 = 不可靠。

**双层缓存**（内存 + 磁盘）：
- 内存缓存：同一次 Claude 会话内，同一个股票查两次不重复请求
- 磁盘缓存（`~/.clawdbot/skills/cn-stock-pro/cache/`）：跨会话有效，报价5min TTL，基本面1h TTL
- 为什么？反爬。如果你 1 分钟内查 10 次茅台，没有缓存 = 10 次 HTTP 请求 → 被 Sina 封 IP。有缓存 → 1 次请求。

**7 个子命令的取舍**：
| 命令 | 为什么有 |
|------|---------|
| `price` | 最基础需求——"现在多少钱" |
| `chart` | 一图胜千言——新手看数字懵，看图秒懂涨跌 |
| `compare` | 选股场景——"茅台和五粮液哪个好"→ 一张图对比 |
| `fundamentals` | 价值投资入门——PE/EPS/ROE 是巴菲特的三个关键词 |
| `earnings` | 风险管理——财报前后是最大波动源，不知道日期 = 裸奔 |
| `history` | 快速复盘——不需要开 app，终端里 10 日走势一目了然 |
| `market` | 大局观——个股再好，大盘崩了一切白搭 |

### 3. scripts/portfolio.py — 投资组合管理

**是什么**：490 行的独立脚本。创建组合、增删持仓、实时盈亏追踪。

**为什么独立于 cnstock 之外**：
- 它有自己的数据模型（`Asset`、`Portfolio` 两个 dataclass）、自己的存储（JSON 文件）、自己的 CLI。
- 不是每次查股票都需要组合功能，分离 = 不污染核心引擎的代码。

**为什么存在**：查股票后下一步自然是"我想记住我买了什么"。从"查"到"管"的闭环。stock-market-pro 没有这个能力，stock-analysis 有但只支持美股。

---

### 4. README.md — 给人看的

**为什么和 SKILL.md 分开**：
- SKILL.md 是给 Claude 读的（机器语言），README.md 是给人读的（人类语言）。
- GitHub 展示的是 README.md，这是用户进入 repo 的第一印象。它写的是"这是什么、怎么装、兼容什么"——面向 action。
- SKILL.md GitHub 也看得见，但那是给"想理解 skill 内部逻辑的开发者"看的。

## 二、用户能获得什么信息（7 条数据管线）

### 管线 1：实时行情 → 不知道价格的人是瞎的
```
price → 当前价、涨跌幅、日内高低、昨收、成交量
```
**为什么有这条管线**：所有投资决策的第一步。"茅台现在多少钱"——这个问题没有答案之前，一切分析都是空中楼阁。成交量尤其关键：价涨量缩=虚涨，价跌量增=真跑。

### 管线 2：K 线图 → 图形比数字快 10 倍
```
chart  → 蜡烛图/折线图 + MA5/20/60 + 成交量柱
```
**为什么有这条管线**：人类是视觉动物。看一列数字"今日最高1215、最低1180、开盘1200、收盘1190"你需要 5 秒在脑内构图；看一根阴线蜡烛你需要 0.5 秒。MA5/MA20/MA60 三条均线 = 短期/中期/长期趋势一目了然。

### 管线 3：多股对比图 → 选股不是在真空中做的
```
compare → 归一化百分比叠加图，所有股票从 0% 起步
```
**为什么有这条管线**：用户说"茅台和五粮液哪个好"——单独看两只的 K 线图没法比，因为茅台 1200 五粮液 76，Y 轴完全不一样。归一化到百分比后"哦茅台 6 个月跌 15%，五粮液跌 30%，茅台更抗跌"——答案瞬间出来。

### 管线 4：基本面 → 你在买"公司"不是在买"代码"
```
fundamentals → PE/PEG/PB/PS + EPS/ROE/ROA + 毛利/运营/净利率
                + 营收增速/盈利增速 + 负债权益比/流动比率/FCF
                + 市值/EV/EV-EBITDA + 股息率 + 52周位置 + 分析师共识
```
**为什么有这条管线**：80% 的散户只看价格不看基本面。这条管线存在的意义就是让你看到"价格背后的东西"。PE 告诉你这个价格贵不贵，ROE 告诉你这家公司赚钱效率怎么样，FCF 告诉你它是不是在烧钱续命。每一条指标都有白话解释——因为术语 = 门槛。

### 管线 5：财报日历 → 最危险的时刻你不应该不知道
```
earnings → 下次财报日 + 倒计时天数 + 风险分级警告
           A股：法定披露截止日倒计时
           美股：历史财报惊喜/失望记录 + 分析师一致预期
```
**为什么有这条管线**：财报前后是股价波动最大的时期。不知道财报什么时候出 = 不知道什么时候可能大幅亏损。倒计时 → 心理准备。"还有 5 天出财报，先别急着买"——这一句话值千金。

### 管线 6：历史趋势 → 过去 10 天发生了什么
```
history → 10 日行情表格：开盘/最高/最低/收盘/涨跌幅/成交量
```
**为什么有这条管线**：10 天 = 两周的交易日。太短看不出趋势，太长又信息过载。10 天刚好够判断"最近是在涨还是在跌"。

### 管线 7：市场温度 → 不看大盘看个股 = 低头走路不看方向
```
market  → VIX 恐慌指数 + SPY/QQQ 趋势 + GLD/TLT/UUP 避险信号 + 上证指数
```
**为什么有这条管线**：2022 年纳指跌 33%，个股再好也大概率跟着跌。VIX >30 = 市场在恐慌，黄金+美债+美元齐涨 = 资金在逃向避险资产。这条管线的作用是让你先看天气再出门。

## 三、竞争差异化：为什么不直接用同花顺/东方财富/Yahoo Finance？

### vs 传统股票 App（同花顺、东方财富、雪球）

| | 传统 App | China-Stock-Market-Pro |
|---|---|---|
| **数据深度** | 极深（Level-2、资金流向、龙虎榜） | 中等（基础行情+基本面+图表） |
| **上手难度** | 高——满屏按钮、指标、弹窗 | 低——说人话就行 |
| **信息筛选** | 你筛选——App 给你所有数据你挑 | Claude 筛选——只给你关键信息+白话解释 |
| **操作方式** | 点、滑、搜、翻 | 说"查一下茅台" |
| **多股对比** | 手动加自选、切来切去 | 一张图全部归一化对比 |
| **个性化** | 自选列表、条件预警 | Claude 根据你的问题定制报告 |
| **学习曲线** | 你需要先学会用软件 | 你会说话就会用 |

**核心差异**：传统 App 是"我给你所有数据，你自己找答案"。这个 skill 是"你问我问题，我给你答案+解释"。

**什么时候用 App**：深度盯盘、做技术分析、看资金流向、设条件单。
**什么时候用这个 skill**：快速了解一只股票、对比几只股票、学习财报基础知识。

### vs Yahoo Finance / Google Finance（网页版）

| | Yahoo Finance | China-Stock-Market-Pro |
|---|---|---|
| A股数据 | ⚠️ 延迟 15min+、缺乏基本面 | ✅ 新浪实时、腾讯基本面字段 |
| 语言 | 英文 | 中文 + 白话解释 |
| 市场覆盖 | 美股为主 | 中美港三地 + Crypto |
| 多股对比 | 手动叠加 | 一条命令出归一化图 |
| 智能化 | 无 | Claude 定制报告 |

### vs stock-analysis skill（另一个 Claude Code skill）

| | stock-analysis | China-Stock-Market-Pro |
|---|---|---|
| **核心能力** | 量化评分引擎（BUY/HOLD/SELL） | 数据展示 + 白话翻译 |
| **分析深度** | 深——8维度加权，舆情/NLP/地缘政治 | 浅——展示数据，不判方向 |
| **A股** | ❌ 不支持 | ✅ 四源降级 |
| **图表** | ❌ 无 | ✅ K线/折线/多股归一化对比 |
| **面向用户** | 美股交易者，懂英文 | 中文小白，A股为主 |
| **代码规模** | 2504 行 | 1667 行 |

两者**互补而非竞争**。stock-analysis 给你"该买还是该卖"的答案，China-Stock-Market-Pro 给你"这公司到底怎么样"的理解。理想情况下，用户两个都装：先用 china 了解公司，再用 stock-analysis 做决策。

## 四、设计取舍：故意不做的事

### 1. 故意不给买卖建议
```
SKILL.md 第10行: "does not give personalized investment advice or buy/sell recommendations"
```
**为什么**：给买卖建议 = 法律责任 + 用户会怪你。"你说茅台会涨我才买的"——这句话是金融科技的死亡诅咒。展示数据 + 解读含义 = 工具；给出买卖信号 = 投顾（需要牌照）。

### 2. 故意不用大模型 API
```
零 API Key，所有数据源免费
```
**为什么**：任何需要 API Key 的功能 = 门槛。散户不会去申请 Yahoo Finance API、不会配密钥。`uv run` 一行命令就能跑 = 零摩擦。

### 3. 故意把脚本写成单体而不是模块化
```
cnstock: 1667 行单文件
```
**为什么**：Claude Code skill 的执行模型是 `uv run --script`，它要求一个文件包含所有 inline 依赖声明。拆成多文件 → `uv run` 支持不了 → 需要 `pip install -e .` → 门槛提高。

**代价是**：代码可维护性差（一个文件 1667 行确实长了）。未来如果超过 2000 行，应该考虑拆成 `fetchers.py` + `cache.py` + `charts.py` + `cli.py`，用 `uv run scripts/cli.py` 当入口，其他文件当普通 import。

### 4. 故意只做浅层数据展示不做分析
**为什么**：chart 技能是"数据展示器"而非"分析引擎"。它不判断方向、不做评分。这是故意的——因为数据准确度是第一位的，分析结论的准确度很难保证。先做好数据，再谈分析。

## 五、总结：一句话说清定位

> **China-Stock-Market-Pro = A股实时数据 + 美股基本面 + 大白话解释 + Claude 定制报告。**
>
> 它不是另一个股票 App，而是一个"会说人话的股票翻译官"——
> 把冰冷的数字翻译成你能听懂的话，把繁琐的操作变成一句话的对话。
